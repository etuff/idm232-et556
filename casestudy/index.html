ReCipe Online Cookbook Web Application

Overview

ReCipe is a custom-built online cookbook web application created for IDM 232 course. The goal of this project was to design and develop a responsive recipe-browsing experience powered by a MySQL database and fully coded with PHP. Over the course of the Alpha, Beta, and Final stages, I designed the interface, documented the system, and built a working platform that allows users to browse,filter search, and explore a full library of recipes.

The project tested both my HTML web development skills that i have known and PHP scripting skills which i learned during this term. At the end, I delivered a fully functioning website with templated layouts, search features that filters, and a clean visual system.

Context and Challenge

Background

The assignment required me to build a fully custom cookbook web application using only PHP, MySQL, CSS, and Git. Pre-written recipes and images were provided, but all design decisions, branding, layout, and coding were done independently.

Problem

The original PDFs were not easy to browse or search and it were not on a central source where everyting would be acessible. I saw this challenge as an opportunity to transform these static recipe files into a cooking website that lets users filter, explore, and read each recipe quickly.

Goals

  • Build a fully responsive cooking website from small to large screens

  • Convert dozens of recipe PDFs into structured excel file and then MySQL data

  • Allow users to browse, filter, and search recipes

  • Deliver the project fully functional by the Final stage

Project Scope

This was a solo project in a span of 10 weeks and i was responsible for;

  • Branding and interface design

  • Front-end development

  • PHP templating

  • MySQL database structure

  • Debugging and optimization

Process and Insight

Research

To understand how users browse recipes, I reviewed patterns from sites like Blue Apron, Half Baked Harvest, and AllRecipes. I focused on:

  • Quick scanning

  • Visual appeal

  • Clear ingredient and instruction hierarchy

Alpha (HTML Web Dev)

HTML based webpage: Alpha HTML webpage

The Alpha phase served as the foundation of the entire project. This stage focused on translating early design decisions into a working HTML prototype. The goal was not full functionality yet, but instead a clear visual and structural representation of the final web application.

I built the initial page layouts completely in HTML and CSS and this included defining:

  • Homepage

  • Responsive grid structure for the recipe cards

  • The hero section with a centered search bar

The focus was on building page templates that would later be converted into dynamic PHP pages, which is why the structure needed to be consistent, reusable, and flexible.

Beta (PHP + MySQL)

In Beta, the project shifted from static HTML to a fully dynamic site. Some things i did for the Beta include;

  • Built a MySQL database to store all recipe data on my localhost

  • Created reusable PHP templates

  • Integrated dynamic content for grids, single recipe pages, and search

  • Copied the database onto Drexel's Server and connected the website to the database using PHP

The goal here was to make the entire project functional enough for submision and feedback.

Final

The Final Build represents the fully realized version of the ReCipe web application. After establishing structure in Alpha and full functionality in Beta, the Final stage focused on completing missing features, integrating all recipes, polishing design details, and ensuring the system performed smoothly on real devices.

Full Integration of PHP & MySQL

The PHP template system created in Beta was fully completed. Every major page pulled live data directly from the MySQL database. Key improvements included;

  • Ensuring all recipe cards populate their images, names, and descriptions.

  • Finalizing logic so all pages respond correctly to various search and filter inputs

  • Reducing repeated code by consolidating functions and templates.

The Solution

The final solution is a fully responsive online cookbook web application that transforms a large collection of static recipe PDFs into an easy-to-browse digital experience. The solution brings together the structure defined in Alpha, the functionality built in Beta, and the polish added in the Final build to create a functional and branded cooking platform.

Here are the core features

Hero section and recipe grid on Hompage

  • A large, clean brand title (“ReCipe”) and a centered Search Bar

  • Minimal distractions and clear visual hierarchy

  • A responsive grid showcases a curated selection of recipes

  • Recipe cards clickable to the detailed recipe page

Search & Filtering System

  • A centered Search Bar and top-right search input accessible from any page.

  • Search results pages populated dynamically through PHP and MySQL

  • Search results filters Recipe name and recipe subtitle

  • A “No Results” screen with clear messaging when search doesn't match any recipe title or subtitle in the database.

Recipe Detailed Pages

  • Shows the finished dish clearly and largely.

  • Provides immediate context on title and description before diving into the steps.

  • Ingredients displayed in a clean, scannable list with consistent formatting.

  • Step-by-step instructions

PHP & MySQL Functionality

  • The homepage recipe cards are generated from the database

  • Each recipe page loads the correct ingredients and steps automatically

  • The search system retrieves matching results in real time

  • Database structure supports all recipes with room for future additions

Results

The final ReCipe web application turned out to be a successful and complete cooking platform that met all the project goals. The biggest improvement was how the site transformed the original PDF recipes into an easy-to-use, searchable experience. Instead of clicking through separate files, users can now browse, search, and open recipes quickly through a clean and organized interface.

If I could change anything with the process on this project, I would certainly start the tasks earlier, such as the search system and the database setup, to avoid feeling rushed near the end. I would also spend more time with the structuring of the database. However, these experiences helped strengthen my process, and the final product shows real growth in both my understanding and coding skills.
